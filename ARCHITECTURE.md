### User Request Journey:

1. **User enters**: `https://saamcabins.com`
   - DNS resolves to `68.178.160.108`
   - Browser connects via HTTPS (port 443)

2. **Nginx receives request**:
   - SSL/TLS handshake (Let's Encrypt certificate)
   - Checks rate limiting (max 10 req/s + burst 20)
   - Applies security headers
   - Routes based on path:
     - `/_next/static/*` â†’ Serves from filesystem (365d cache)
     - `/images/*` â†’ Serves from public folder (30d cache)
     - Everything else â†’ Proxy to PM2

3. **PM2 load balances**:
   - Distributes request to one of N Next.js instances
   - Uses round-robin or least-connection algorithm
   - Maintains connection pooling

4. **Next.js processes request**:
   - App Router determines route
   - Server-side rendering (SSR) or Static generation
   - Fetches data from CMS if needed (Sanity)
   - Optimizes images on-the-fly
   - Generates HTML response

5. **Response flows back**:
   - Next.js â†’ PM2 â†’ Nginx â†’ User
   - Nginx applies Gzip compression
   - Adds caching headers
   - Sends to client

**Total time**: ~50-200ms (depending on page complexity)
